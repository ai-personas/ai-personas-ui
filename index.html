PersonaOS Discovery

This page discovers at runtime: it bootstraps from .well-known, does a Kademlia DHT provider lookup, then resolves and cryptographically verifies every record live in your browser (Ed25519) before showing it — across the internet (.well-known + gossip + DHT) and intranet (mDNS) planes, access-gated discover < read < write < admin.

Discovery log

    How this discovers (not a directory dump)

    Records are not listed inline. The browser fetches a DHT provider index of opaque key → record_url pointers, then resolves each record and verifies its Ed25519 signature against the kernel's published key before trust. A failed signature is dropped. Add a peer kernel's URL above and it is discovered and verified the same way — this portal is generic for any PersonaOS kernel. True multicast mDNS / libp2p Kademlia run in the native runtime; the browser profile verifies over HTTP(S) (09_PROTOCOLS §3H.3).

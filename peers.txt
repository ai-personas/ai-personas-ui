# At boot the portal fetches this file, then for every URL below it bootstraps
# that node's `.well-known/personaos-discovery.json`, runs the Kademlia-style
# provider lookup, resolves each record and **Ed25519-verifies it in-browser**
# (exactly like adding `?peer=<url>` or the "＋ PEER" button). Blank lines and
# lines starting with `#` are ignored. Trust is the signature, not the host —
# so a node reached through a relay/tunnel is verified the same as any other.
